# Boolean List Hierarchy

## 节点信息

- 节点 ID：`BooleanListHierarchy`
- 显示名称：`Boolean List Hierarchy`
- 分类：`utils/logic`

## 功能

`Boolean List Hierarchy` 在一个节点内维护最多 64 个动态布尔输出，并支持一层父子依赖：

- 父项为 `false` 时，其子项立即变为 `false`，且子项布尔开关被禁用。
- 父项恢复为 `true` 后，子项开关恢复可用，但不会恢复关闭前的值。
- 子项不会反向修改父项。
- 每个条目使用稳定 ID；层级编辑或排序后，已有输出连线仍跟随原条目。

## 输入与输出

- `config_json`：隐藏的内部配置输入，由前端编辑器维护并随工作流序列化；通常无需手动修改。
- `Boolean 1` 至 `Boolean 64`：固定后端输出槽。当前可见条目按界面顺序映射到前 N 个输出，其余输出为 `false`。

条目名称只影响显示，不影响稳定 ID。重命名、同级移动、缩进或提升条目时，前端会按稳定 ID 重排连接，避免连线错绑到其他布尔项。

## 行级 ICON

- 新增子项
- 同级上移、下移
- 缩进到前一个父项；已有子项的父项不可缩进
- 将子项提升为根项
- 删除条目；删除父项时级联删除其子项

工具栏提供新增根项。节点始终至少保留一个根项，达到 64 项后停止新增。

父项关闭时只禁用子项的布尔开关；名称编辑、排序、提升和删除仍可使用。

## 使用步骤

1. 点击工具栏新增根项，或点击某个父项的“新增子项”图标。
2. 编辑条目名称并设置布尔值。
3. 使用上移、下移、缩进和提升图标组织一层父子关系。
4. 从对应输出连接需要接收布尔值的节点；后续重排不会改变条目的稳定绑定。

父项从 `true` 变为 `false` 时，其直接子项会立即写为 `false`。父项重新开启后，子项只恢复可编辑状态，不恢复关闭前的旧值。

## 兼容性

旧版 `{label, value, level}` 配置会在加载时迁移为 `{id, label, value, parent_id}`。孤立子项、指向子项的二级嵌套和无效父项会自动提升为根项。节点 ID、`boolean_list_items` 属性名和最多 64 个后端输出保持不变。

较新的工作流通过隐藏的 `config_json` 传递当前界面状态；没有该输入的旧工作流仍会从 `properties.boolean_list_items` 恢复配置。

## 测试覆盖

- 后端：旧配置迁移、父子级联、无效层级修复、64 路输出顺序和旧工作流回退。
- 前端：增删、排序、缩进/提升、容量限制，以及重排后按稳定 ID 保持连线。
