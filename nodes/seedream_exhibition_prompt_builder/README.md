# Seedream Exhibition Prompt Builder

## 节点信息

- 节点 ID：`SeedreamExhibitionPromptBuilder`
- 显示名称：`Seedream Exhibition Prompt Builder`
- 分类：`Seedream/Prompt`
- 输出：`prompt: STRING`

## 用途

该节点用于替代展厅工作流中由 `GPT Image Style Prompt Preset`、多个 `LazySwitchKJ` 和 `Text Concatenate` 组成的提示词拼接段。它会根据主题风格、参考图用途、色彩、语气、材料补充描述和 5 个布尔条件，生成更适合 Seedream 4.0/4.5 的自然语言结构化提示词。

当前节点只面向展厅写实渲染效果图，不做 GPT Image 双模式，也不修改旧 `GPT Image Style Prompt Preset` 的输出逻辑。

## 输入

### 固定输入

- `style_id`：主题风格。前端下拉显示中文标签，数据来自 `web/styles.json`，例如 `航天科技`、`商务`、`党建`。
- `tone`：画面语气，可选 `标准`、`明亮`、`稳重`、`高级`、`活泼`。
- `primary_color`：主色 Color Picker。内部解析 Hex，但最终提示词默认输出自然语言色彩，例如 `明亮科技蓝`、`主题红`、`金色点缀`。
- `secondary_color`：辅色 Color Picker。内部解析 Hex，但最终提示词默认输出自然语言色彩，例如 `冷白色`、`浅灰过渡色`、`深灰边界色`。
- `base_prompt`：不使用主题模板时的基础场景描述。
- `additional_details`：材料、工艺、灯光、顶地墙等补充描述。空字符串会被忽略；单独的 Hex 值会被过滤。

### 布尔输入

5 个布尔项均固定暴露为可连线输入：

- `use_theme_template`：是否使用 `web/styles.json` 中的主题标签作为风格入口。
- `use_space_reference`：是否把参考图作为展厅空间布局与建筑结构依据。
- `include_people_placeholder`：是否将参考图中的占位人物或示意人形替换为写实游客。仅在 `use_space_reference=true` 时生效。
- `use_element_reference`：是否把参考图中的物品或元素作为元素参考。
- `lock_edit_region`：是否要求元素保持原位置。仅在 `use_element_reference=true` 时生效。

当前版本不提供 `visible_text` 输入，不会自动把补充描述包装为画面可见文字。需要在画面中出现的标语、标题或展板文字，建议作为独立提示词节点处理，并只对真实可见文字使用双引号。

## 拼接逻辑

节点按固定顺序组织提示词，避免逗号硬拼：

1. 任务意图：创建展厅写实渲染效果图，用于展厅设计方案呈现。
2. 主题或基础描述：`use_theme_template=true` 时使用所选风格标签；否则追加 `base_prompt`。
3. 空间参考：启用空间参考时，明确参考图用于空间布局、建筑结构、墙顶地、动线和核心结构位置。
4. 人物参考：启用空间参考且启用人物替换时，将占位人物或示意人形替换为自然站立、比例合理的真实游客，并明确不保留占位符外观。
5. 元素参考：启用元素参考时，把参考图中的物品或元素作为整体元素参考；锁定时保留原位置，否则放在合理位置。
6. 材料补充：追加 `additional_details` 中的有效描述。
7. 色彩控制：输出自然语言主色和辅色用途，强调中性材质基底、局部信息面、边界、过渡、灯光层次和留白。
8. 语气与构图：追加 `tone` 对应的光线、对比度、色彩和构图约束。
9. 保护约束：保持现有空间结构和展陈元素，不额外添加未要求的主展品、视觉焦点或大型装置。

## 色彩策略

节点保留 Color Picker 输入，但提示词中默认不输出 `#RRGGBB`。颜色会转换为更自然的中文语义：

- 接近白色：`冷白色`
- 接近黑色：`深黑色`
- 灰阶：`浅灰过渡色`、`中性灰`、`深灰边界色`
- 蓝色系：`科技蓝`、`明亮科技蓝`、`高饱和科技蓝`
- 青色系：`冷光青蓝`、`柔和青色`
- 红色系：`主题红`、`稳重红色`
- 黄色系：`金色点缀`、`柔和暖黄色`
- 绿色系：`活力绿`、`柔和绿色`
- 紫色系：`科技紫`、`深紫色`

色彩段落会强调主色只用于局部信息面、屏幕内容、发光线条和导视强调；辅色用于留白、边界、过渡面和灯光层次。当前表达避免使用固定的“分段模块化色彩节奏”，改为更灵活的“墙面色彩与材质层次自然变化”，以减少对风格形式的硬限定。

## 主题模板

主题数据来自 `web/styles.json`：

- `label`：前端下拉显示名称。
- `subject`：主题主体，可供其他节点或旧节点使用。
- `prompt`：主题视觉描述。当前 Seedream 节点不会直接把长模板全文拼入提示词，只用 `label` 生成简洁风格入口，避免模板内容过长稀释风格或把主题图案扩展成整墙铺陈。
- `thumbnail`：前端缩略图选择器使用的图片文件名。

## 前端行为

`web/prompt_preset.js` 同时支持：

- `GPTImageStylePromptPreset`
- `SeedreamExhibitionPromptBuilder`

两个节点都会使用同一套缩略图风格选择器。`style_id` 在前端显示中文标签，同时后端仍兼容英文 style key。

## 示例输出

选择 `航天科技`、主色 `#567DF0`、辅色 `#FFFFFF`，并启用主题模板时，输出会类似：

```text
创建一张航天科技风格的展厅写实渲染效果图，用于展厅设计方案呈现。以低饱和中性色和浅色材质为基底。主色调为明亮科技蓝，用于局部信息面、屏幕内容、发光线条和导视强调，辅色为冷白色，用于留白、边界、过渡面和灯光层次。墙面色彩与材质层次自然变化，主色信息面之间穿插中性墙面、材质分割和留白，避免整墙连续色块。画面光线自然柔和，对比度适中，色彩还原准确，构图均衡稳定。保持现有空间结构和展陈元素，不额外添加未要求的主展品、视觉焦点或大型装置。室内建筑摄影视角，构图清晰，空间层次明确，整体克制、通透、有节奏。
```
